# 中1テストキット 設計書

- **日付**: 2026-05-08
- **ステータス**: ドラフト（ユーザーレビュー待ち）
- **対象テスト機会**: 1学期中間テスト（2026年5月下旬〜6月上旬）— 間に合わなければそれ以降のテストへ繰り越し

---

## 1. プロダクト概要

中学1年生の中間テスト対策に特化した5教科対応のPWA。中1の「初めての定期テスト」というタイミングに焦点を当て、保護者が課金し、子ども本人が日常的に使うことを想定する。

- **名称（仮）**: 中1テストキット
- **形態**: PWA（Webアプリ、ホーム画面追加可）
- **価格**: 980円買い切り（購入から30日アクセス）
- **対象学年**: 中学1年生
- **対応範囲**: 5教科（国語・数学・英語・理科・社会）
- **MVP対象範囲**: 中1学習指導要領の共通範囲（学校・教科書出版社別カスタマイズはしない）

## 2. 動機と背景

### 入口戦略
中1の最初の定期テストは「初めての定期テスト」であり、保護者の関心と不安が一年で最も高まるタイミング。AIや学習ツールに頼りたくなる瞬間で、購入意思決定がもっとも進みやすい。

### 副次目標：貧困対策
塾に通えない家庭の子どもにも、低価格で5教科対策を届けることを副次目標とする。価格設定（980円）自体が同カテゴリ最安水準で、実質的な貧困対策となる。寄付モデルやNPO連携はV1では実装せず、ユーザー数が一定規模に達してから検討する（V2以降）。

### 学習体験の差別化
既存サービス（スタディサプリ、Monoxer等）と比べ、「テスト1回」に特化した買い切り価格で、テスト直前期の集中対策に最適化する。

## 3. 目標と非目標

### 目標
- 中1の最初の中間テスト対策に必要な5教科の頻出範囲を、買い切り980円で提供
- 子ども本人が毎日触りたくなるシンプルなUX
- AI質問機能で「わからない」を即座に解消できる学習体験
- 学習指導要領ベースで「公的根拠のある」コンテンツを提供

### 非目標
- 学校・教科書出版社ごとのカスタマイズ（V2以降も実装しない）
- 中1中間以外のテスト・学年への対応（将来別商品で展開）
- 教師・塾向け管理機能（B2B展開は別途検討）
- 動画コンテンツ（テキスト＋AIで完結）

## 4. ターゲットユーザー

### 主要ユーザー（毎日触る）
- 中学1年生本人
- スマホまたはタブレットで使用（家庭のPCでも可）

### 課金ユーザー（購入の意思決定者）
- 中1の保護者
- 塾通いが厳しい家庭、または塾の補助として使いたい家庭

### 保護者の関与
保護者は購入と進捗の確認のみ。子ども専用のダッシュボードや学習体験を妨げない設計とする。

## 5. ユーザージャーニー

1. **流入**: 保護者または子どもが「中1 中間テスト 対策」「初めての定期テスト」等で検索 → ランディングページへ
2. **購入**: ランディングページで内容確認 → Stripe Checkout で980円決済 → 購入完了メールでマジックリンク受信
3. **初回ログイン**: マジックリンククリック → 認証 → 30日カウントダウン開始
4. **初回セットアップ**: 教科書出版社、テスト日を入力（学校名は収集しない）
5. **ホーム**: 5教科タイル＋テスト●日前カウンター＋全体習熟度
6. **教科選択 → トピック一覧**: 各トピックに「未習・学習中・習得」のステータス
7. **トピック内ループ**:
   - **解説**（学習指導要領ベース、3-5分で読める）
   - **ドリル**（10-20問、連続◯問正解で「習得」マーク）
   - **AIに質問**（常時オープン、文字または写真で）
8. **直前期**: テスト1週間前に「予想模試」が解放される（5教科横断、時間制限付き）
9. **テスト後**: 自己採点入力で実績ログ → 期末や次中間に向けたリピート購入導線

## 6. 機能仕様

### 6.1 学習ループ（粒度：ハイブリッド）
- **トピック単位** で「解説 → ドリル → AI質問」の3ステップを繰り返す
- **予想模試（4ステップ目）** はテスト1週間前に解放される全教科共通の独立体験

### 6.2 解説
- 学習指導要領の「目標」と「指導内容」に沿って、頻出ポイントを3-5分で読めるテキストに圧縮
- マークダウン形式で、見出し・箇条書き・例・公式を含む
- 各解説の最後に「ドリルへ進む」CTA

### 6.3 ドリル
- 1トピックあたり10-20問
- 問題形式: 4択、短答、計算入力（教科に応じて）
- 連続◯問正解（例: 5問）で「習得」マークを付与
- 間違えた問題は理由つきで即フィードバック
- 「この問題をAIに聞く」ボタンを常時表示

### 6.4 AI質問
- 入力: 文字メイン、画像アップロード可（R2に保存）
- モデル: Claude Haiku 4.5（デフォルト）
- レート制限: 1日30問/ユーザー（KVカウンタ）
- ストリーミング表示でUX向上
- プロンプトキャッシュで教材コンテキストを使い回し

### 6.5 予想模試
- テスト1週間前に解放（テスト日 - 7日でアンロック）
- 5教科横断、時間制限付き（科目ごとに50分目安）
- 学習指導要領＋過去出題傾向ベースの全国共通版
- 終了後、教科別スコア＋苦手分野フィードバック

### 6.6 進捗ダッシュボード
- 5教科×トピック別の習熟度（未習・学習中・習得）
- 累計学習時間、解いた問題数、AI質問回数
- 残日数（30日カウントダウン）

### 6.7 購入と認証
- Stripe Checkout で980円決済（一括）
- 決済完了後、メールアドレス宛にマジックリンク送信（Resend）
- ログインはマジックリンクのみ（パスワード不要）
- アクセス権は購入から30日

### 6.8 期間切れ後の挙動
- 30日経過後、学習データは保持（再購入で復元）
- ログインは可能、ただし学習画面はロック
- 「次のテストに向けてリピート購入」導線を表示

## 7. 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Next.js 16（App Router）+ Turbopack |
| PWA | manifest.json + Service Worker（next-pwa等） |
| ホスティング | Cloudflare Pages |
| バックエンドAPI | Cloudflare Workers + Hono |
| DB | Cloudflare D1（SQLite互換） |
| KV | Cloudflare KV（AIレート制限カウンタ） |
| ファイル | Cloudflare R2（質問用画像） |
| 認証 | メールマジックリンク（Resend経由） |
| 決済 | Stripe Checkout（直接、AppStore非経由） |
| AI（質問応答） | Claude Haiku 4.5 |
| AI（コンテンツ事前生成） | Claude Sonnet 4.6 |
| 監視 | Cloudflare Analytics + Sentry（エラー監視） |

## 8. データモデル

```
users
  - id (UUID)
  - email (string, unique)
  - purchased_at (timestamp, ISO 8601)
  - expires_at (timestamp, ISO 8601, purchased_at + 30 days)
  - textbook_publisher (string, 任意)
  - test_date (date, ISO 8601)
  - created_at, updated_at

subjects
  - id (PK)
  - name (国語/数学/英語/理科/社会)
  - order (sortキー)

topics
  - id (PK)
  - subject_id (FK -> subjects)
  - name
  - order
  - curriculum_ref (学習指導要領の参照ID)

lessons
  - id (PK)
  - topic_id (FK -> topics)
  - content_md (Markdown本文)
  - estimated_minutes

questions
  - id (PK)
  - topic_id (FK -> topics)
  - type (multi_choice / short_answer / numeric)
  - question_text
  - choices (JSON, 任意)
  - answer (string)
  - explanation (string)
  - difficulty (1-3)

attempts
  - id (PK)
  - user_id (FK)
  - question_id (FK)
  - user_answer (string)
  - is_correct (bool)
  - attempted_at (timestamp, ISO 8601)

topic_progress
  - user_id (FK, PK)
  - topic_id (FK, PK)
  - mastery_level (0-3: 未習/学習中/習得/復習推奨)
  - consecutive_correct (int)
  - last_studied_at (timestamp, ISO 8601)

ai_chats
  - id (PK)
  - user_id (FK)
  - topic_id (FK, 任意)
  - prompt (text)
  - response (text)
  - tokens_in (int)
  - tokens_out (int)
  - image_url (string, 任意)
  - created_at (timestamp, ISO 8601)

ai_usage_daily（KVに保存、DBは集計用）
  - user_id (FK)
  - date (ISO 8601 date)
  - count

mock_exams
  - id (PK)
  - user_id (FK)
  - started_at (timestamp, ISO 8601)
  - finished_at (timestamp, ISO 8601, 任意)
  - score (int, 任意)

mock_exam_items
  - exam_id (FK, PK)
  - question_id (FK, PK)
  - user_answer (string)
  - is_correct (bool)

referrals (V2用、空テーブル予約)
  - id (PK)
  - inviter_id (FK -> users)
  - invitee_id (FK -> users, 任意)
  - gifted_subject_id (FK -> subjects, 任意)
  - redeemed_at (timestamp, ISO 8601, 任意)
  - created_at
```

### 設計上の注意
- `users` テーブルから `school` 関連カラムは除外（学校別カスタマイズは実装しない）
- `mock_exams` から `school_specific` 列も除外
- `referrals` テーブルは V1 では使わないが、スキーマだけ用意

## 9. API設計（概要）

### 認証
- `POST /api/auth/magic-link` — メールアドレス受け取り、マジックリンク送信
- `GET /api/auth/verify?token=...` — トークン検証、セッション発行

### 購入
- `POST /api/checkout/session` — Stripe Checkout Session作成
- `POST /api/webhooks/stripe` — 決済完了Webhook、users レコード作成

### 学習
- `GET /api/subjects` — 5教科一覧
- `GET /api/subjects/:id/topics` — 教科のトピック一覧（進捗付き）
- `GET /api/topics/:id/lesson` — 解説本文
- `GET /api/topics/:id/questions` — ドリル問題セット
- `POST /api/attempts` — 回答記録、正誤判定、習熟度更新

### AI質問
- `POST /api/ai/ask` — Claude Haiku 4.5 にストリーミング問い合わせ
- レート制限ミドルウェアで日次30問チェック

### 模試
- `GET /api/mock-exam/availability` — 解放可否を返す
- `POST /api/mock-exam/start` — 模試開始
- `POST /api/mock-exam/submit` — 模試提出、採点

### 進捗
- `GET /api/progress/summary` — 全体習熟度サマリ
- `GET /api/profile` — ユーザープロフィール（残日数、購入履歴）

## 10. コンテンツ生成パイプライン

### 事前生成
1. 学習指導要領テキスト＋教科書の章立てサマリをプロンプトに固定
2. Claude Sonnet 4.6 で5教科×頻出トピック上位（教科ごとに5-8トピック）×解説＋問題＋解説をJSON生成
3. 生成結果を人間（開発者本人）が抜き取りレビュー
4. D1 に投入

### 品質管理ループ
- 全問題に「この問題おかしい？」報告ボタン
- 報告は管理画面（簡易、Cloudflare Workers + 簡易認証）で確認・修正
- 修正は次回ログイン時に反映

### V1.1 以降
- 監修者（元教師・塾講師）による定期レビュー導入
- 修正後コンテンツへの段階的差し替え

## 12. セキュリティ

- マジックリンクトークンは1回限り、有効期限15分
- Stripe Webhook の署名検証
- AI質問の入力サニタイズ（プロンプトインジェクション対策）
- R2画像アップロードはサインドURL経由
- 子ども向けサービスとして、不適切コンテンツのモデレーション（Claude側のセーフティとプロンプト制約）

## 13. リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| LLM生成コンテンツの誤答 | 信頼喪失 | 報告ボタン＋即修正ループ、リリース前に開発者レビュー |
| Stripe審査遅延 | リリース延期 | Stripe事業者登録を最優先で着手 |
| AI質問の暴走コスト | 赤字化 | 日次上限＋Haikuベース＋プロンプトキャッシュ |
| 中1の継続率低下 | リピート購入されない | ゲーミフィケーション要素（連続正解、習得バッジ）、進捗の可視化 |
| 保護者からの「内容が薄い」クレーム | 返金、評判低下 | 学習指導要領ベースを明示、ローンチ前にコンテンツ網羅率を計測 |
| iOS PWA決済の制約 | 決済できない | Stripe Checkout経由なら問題なし、StoreKitは使わない |

## 14. V2に向けた構造的余地

### 友達紹介機能（V2の核）
- 自分の学習量（解いた問題数、連続日数）に応じて「友達への1教科無償アクセス権」を獲得
- 紹介された側は、選んだ1教科を無償で30日間使える
- データ構造: `referrals` テーブル予約済み

### 実装しない（永久に）
- 学校別カスタマイズ
- 学校別予想問題・学校別模試
- 教師・塾向け管理機能（別商品で展開）

## 15. 主要なオープン課題

1. **コンテンツの初期投入量**: 5教科×何トピック×何問が現実的か。事前生成と人間レビューの工数を見積もる必要あり。
2. **AI質問の応答品質ガードレール**: 中学生が誤った情報を信じてしまうリスクをどう抑えるか。プロンプト設計の検証が必要。
3. **保護者向け説明文**: 「AI生成コンテンツであること」「継続改善中であること」をどう伝えるか。LP上の透明性。
4. **テスト日が学校ごとに違う問題**: ユーザーが入力するテスト日を信じる前提でいいか（学校別データを持たないため）。
5. **30日経過後のデータ保持期間**: 永続保持か、6ヶ月で自動削除か。GDPR/個人情報保護観点。

## 付録: 用語

- **学習指導要領**: 文部科学省が定める教育課程の基準。中学校は文部科学省ウェブサイトに公開されている。
- **PWA**: Progressive Web App。ブラウザベースだがホーム画面追加・オフライン動作が可能。
- **マジックリンク**: パスワードレス認証。メールに送信されたリンクをクリックするだけでログインできる。
